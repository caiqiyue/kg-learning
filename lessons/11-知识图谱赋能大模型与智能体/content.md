# 11 知识图谱如何帮助 LLM 与 Agent

## 引言

LLM 会说话，但它不天然拥有企业实时知识、权限边界、事实溯源和稳定记忆。知识图谱可以成为 LLM/Agent 的外部知识层。

## KG 如何帮助 LLM

知识图谱可以提供：

- **事实约束**：回答必须基于图中事实。
- **多跳路径**：让模型看到实体之间的关系链。
- **来源追踪**：每个实体可回到 chunk 和文档。
- **结构化查询**：用 Cypher 回答计数、过滤、路径类问题。
- **混合检索**：结合向量、全文、图邻域和社区摘要。

这能减少幻觉，但不能自动消灭幻觉。模型仍然需要被约束：只使用检索上下文、引用来源、不能编造关系。

## KG 如何帮助 Agent

Agent 需要记忆和工具。知识图谱适合做：

- 项目知识库：文件、函数、需求、issue、PR、人员关系。
- 客户知识库：客户、订单、产品、工单、历史事件。
- 研究知识库：论文、作者、方法、数据集、实验结果。
- 行动前检查：某个操作会影响哪些实体和关系？

Agent 可以把 Cypher 查询作为工具，也可以把图谱写入作为长期记忆。但写入要有审核机制，否则 Agent 会把错误记忆永久化。

## 一个 Agent 场景

发布前，Agent 收到任务：

```text
检查登录功能改动会影响哪些客户、文档和历史故障。
```

它可以调用图谱工具：

1. 查询 `登录功能` 相关实体。
2. 沿 1 到 3 跳找到相关服务、代码文件、工单、客户。
3. 返回来源 chunk 和文档。
4. 只读展示风险，不直接修改图谱。

这比让 Agent 在一堆文档里盲搜更可靠，因为图谱明确保存了“功能、服务、文件、客户、故障”的关系。

## Agent 图谱工具设计

一个可靠工具应该声明：

- 输入参数。
- 查询范围。
- 是否允许写入。
- 返回字段。
- 权限要求。
- 失败时如何处理。

知识图谱工具不是“让 Agent 随便执行 Cypher”。生产系统要限制可执行模板，避免越权查询和破坏性写入。

## 小结

知识图谱让 LLM/Agent 从“会生成文本”走向“能查询事实、追踪来源、理解关系、规划行动”。它最适合做企业 AI 的记忆层、证据层和工具层。
